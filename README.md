**Context:** This file is from inside Booko, which is a dynamic pricing platform for service businesses, in which I worked for as a founding engineer. The platform runs A/B tests where some customers are offered a booking incentive (a reward of some dollar amount or points) and others are placed in a control group with no incentive. Across thousands of time slots, this generates outcome data which businesses can use, such as which slots got booked, what incentive was offered, or whether the slot was control or treatment. The pricing engine needs to know how much extra occupancy you actually get from a $25 incentive vs. a $50 incentive vs. a $100 incentive. That's the price elasticity question this file that I built answers.

**What it does:** The code walks through historical booking outcomes, splits them into control and treatment groups, and buckets the treatment slots by incentive value. For each bucket, it calculates the lift in occupancy over the control average and gates on sample size so that it doesn't trust tiny groups. Those lift estimates are then fit with a monotonic isotonic regression, which produces a non-decreasing curve, building a belief that "more incentive can't predict less lift," even when a noisy bucket would otherwise suggest it. A final helper fills in the gaps between the dollar amounts that were actually tested. So even though the A/B tests only measured outcomes at every 25 dollars, the pricing engine can still ask "what lift should I expect at $37?" and get a real answer from the curve between those points.
